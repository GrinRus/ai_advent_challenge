## AIDD:SPEC_PACK
Updated: 2026-03-18
Spec: aidd/docs/spec/TST-001.spec.yaml (status: draft — OUTDATED)

- Goal: Implement live RBAC enforcement on mutation APIs with auditable role operations and SSE-based frontend synchronization
- Non-goals:
  - UI for role management (assumes existing admin interface)
  - Migration of historical access logs
  - Multi-tenant RBAC scope
  - WebSocket (Spec says WebSocket, but Plan uses SSE — follow Plan)
- Key decisions:
  - Backend: Custom WebFlux filter for RBAC (no Spring Security)
  - Auth: X-Profile-Key header-based identity extraction
  - Frontend: React + Zustand (not Redux as in Spec)
  - Session Sync: SSE (Server-Sent Events) — NOT WebSocket, NOT Polling
  - Cache: Caffeine with 5min TTL
  - Audit: PostgreSQL table `rbac_audit_log` with JSONB metadata
- Risks:
  - R-001: Profile service unavailable → Circuit breaker, fail-closed, cache roles
  - R-002: Cache stale roles after assignment → Cache invalidation on role change
  - R-003: SSE connection memory leak → Proper cleanup on disconnect
  - R-004: Audit log write failures → Async logging, error queue
  - R-005: RBAC filter performance degradation → Caffeine cache, async audit
  - R-006: Breaking change to existing APIs → Feature flags, audit-only mode first
  - R-007: Component scan misses RBAC beans → Add to all profile configs
  - R-008: Missing profile service integration → Define contract or implement local role storage
